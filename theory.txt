a.	Que permet de faire la commande git pull ? 
  > La commande git pull permet de mettre à jour le dépôt local (prendre les changements) sur la branche cible en faisant réference au dépôt distant.

b.	Que permet de faire la commande git clone ? 
  > La commande git clone permet de faire une copie en local du dépôt distant et donne une possibilité de se relier.

c.	Expliquer ce que fait un « fork » ?
  > Un « fork » est une copie en remote ou à distant d'un autre dépôt en remote pour pouvoir modifier sans faire de changement sur le dépôt d'origin directement.

d.	Expliquez brièvement l’utilité d’une pull request (PR) ? 
  > Une pull request (PR) est une demande de fusion de sa branche sur une autre branche c'est à dire faire une requete d'ajout de changement venant de sa branche.
